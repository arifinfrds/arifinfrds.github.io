Architecture

Structured with MVVM and Clean Architecture — keeping Presentation, Domain, and Data layers independent, making the app maintainable and the logic independently testable.

MVVM Clean Architecture
🌐
Networking — Moya & GraphQL

API calls managed through Moya for a type-safe REST layer. GraphQL used for flexible queries on health data, test results, and appointment records — reducing unnecessary data transfer.

Moya GraphQL
🔁
Reactive — RxSwift

RxSwift drives the reactive data flow between ViewModels and views — managing async event streams for booking flows, test result updates, and appointment state changes.

RxSwift
🧪
Testing — TDD & Unit Tests

Applied Test-Driven Development (TDD) as the primary development approach. Unit tests cover ViewModels, Use Cases, and repository implementations — particularly important in a healthcare context where data accuracy is critical.

Unit Testing TDD

Structure

Clean Architecture Layers

Layer separation ensures business rules stay independent of UI and data concerns — critical for a healthcare app where reliability matters.

Presentation Layer

Views & ViewModels. RxSwift-driven state. No business logic.

Domain Layer

Use Cases & Entities. Pure Swift — framework-independent. Fully unit-testable.

Data Layer

Repositories, Moya REST + GraphQL services. Maps health data DTOs to domain models.
